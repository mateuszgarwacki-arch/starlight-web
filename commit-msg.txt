Fix blank-page print regression on traveller + other print pages

My handover print CSS added an aggressive whitelist rule:

  body > *:not(.handover-root):not(.handover-print-area) {
    display: none !important;
  }

This was meant to hide the dashboard shell when printing a handover,
but it applied to EVERY print call — so clicking Print on the
traveller, load list, or any other print surface hid the actual
content and produced a blank page.

The existing traveller @media print block at the top of globals.css
already handles dashboard chrome hiding via `nav, .sidebar, header,
.print\:hidden`. My rule was both redundant and harmful.

Removed the whitelist. Handover print still works because:
- Chrome hiding comes from the existing traveller rule (which
  applies globally, same as all other print surfaces).
- The handover-specific `.handover-toolbar { display: none }` rule
  still handles the handover's own on-screen toolbar.

Added a comment above the handover block warning future edits
against re-adding the whitelist pattern.
