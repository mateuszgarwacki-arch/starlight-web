feat(m/me): 10-day overview with backfill of missing time entries

Freelancers can now see their last 10 calendar days (including weekends)
as collapsible day rows above "Recent Entries". Each row shows the date,
weekday, and total hours for that day. Days that have entries but total
under 9h get a red total — the visual signal that something's missing.
Days with zero entries stay neutral (could be a weekend or day off).

Tapping a row expands to show the individual entries (WO time entries +
ad-hoc tasks merged). Each shows description, job number, hours, and a
"Pending" badge if it's a backfill awaiting PM review. At the bottom of
the expanded row, an "Add entry for [date]" button opens a LogSheet
pre-set to that date.

Backfill flow:
- Reuses the existing LogSheet component with a new `defaultRoutedWo`
  prop (for future edit support) and a `woPickerLabel` override so the
  picker reads "Work Order (required)" rather than "Route to WO
  (optional)".
- Handler rejects submit without a WO chosen.
- Synthetic timestamps: start at 09:00 local on the target date, end at
  start + hours. PM can adjust if needed.
- Hourly rate snapshot pulled from tbl_freelancers (matches the existing
  timer-log pattern).
- Inserted via auditedInsert so the change is tracked.
- flag_note prefix "[Backfill]" makes the entry surface in
  /review/timesheets for PM approval.
- notify() fires wo_flagged severity warning so the PM gets a
  real-time bump.

Routed tasks are excluded from the 10-day count because each routing
also creates a tbl_wo_time_entries row — counting both would double up
the day's total.

Data fetch: added two new queries to the existing Promise.all (one for
WO entries in the window, one for non-routed tasks in the window). The
WO/job lookup was refactored to span both the existing recent-20 list
and the new 10-day window in a single fetch — no extra round trips for
the common case.

Edit-existing-entry is deferred. The state machine has a placeholder
"edit" mode (typed in `backfillTarget`) for the follow-up.
