feat(labels): per-WO 2x1 item labels for Zebra GT800 (S53)

New /labels?woId= print route — one 2x1in label per job item linked to
the WO. Label face: JOB# . WO#, item description (large, human-readable),
and a QR to /m/wo/{woId} so a phone camera opens the WO page.

Offline-first: finish_required is printed ON the face (the painter's
primary need is never hidden behind a network call), and only when the
WO carries a PAINT or COVER activity and the field is set. The QR is the
"show me the drawing" bonus layer.

- quantity -> copies, capped at 30; measured-material overflow (e.g. sqm
  of floor covering) prints a single "Qty N" label instead of a runaway
  spool. k/n stamped when copies > 1.
- junction dedup via .in() (tbl_jobitem_workorder has no unique key).
- PrintLabelsButton (Tags icon) sits beside the traveller printer icon
  on each WO row in the work-orders panel.
- Manual print (not auto): GT800 is a different printer from the A4
  default, so the user picks it in the dialog. @page size 2in 1in.

No schema change — finish_required already on tbl_job_items; reuses the
existing qrcode.react dependency.
