Hide dashboard chrome on print (sidebar + topbar + recent-jobs-strip)

The handover print preview was leaking three bits of dashboard chrome
onto every printed page: the sidebar, the top bar with ViewSwitcher,
and the RecentJobsStrip under it.

Root cause: my existing @media print CSS hides `nav, .sidebar, header`
but the sidebar uses `<aside>`. The traveller doesn't suffer this
because /traveller sits outside the (dashboard) route group and has
no dashboard layout — but anything inside (dashboard) that prints
(handover, future reports) has been leaking the whole shell.

Fixes, targeted via Tailwind `print:hidden` so we don't need CSS
selector games that could over-match:

- components/sidebar.tsx — add print:hidden to the <aside> root
- components/pm-sidebar.tsx — same treatment for the PM-view sidebar
- app/(dashboard)/layout.tsx:
  - <main> gets print:ml-0 so content sits at page edge (no offset
    once the sidebar is hidden)
  - Top bar div gets print:hidden
  - RecentJobsStrip wrapped in print:hidden
  - Inner content container gets print:p-0 print:max-w-none so A4
    layout isn't constrained by dashboard padding

This is a global improvement. Any future printable page inside
(dashboard) will inherit clean print chrome handling without needing
page-specific CSS.
