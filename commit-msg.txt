feat(labels): generate hard-sized 50x25mm PDF for the GT800 (S53)

Chrome's HTML print won't honour the die-cut label pitch — output
straddled two physical labels regardless of @page size. Switched the
"Print labels" button to build a PDF whose pages are exactly 50x25mm
(landscape) and hand it to the existing ZDesigner GT800 driver, the same
reliable path MS Access uses. No Browser Print install needed.

- jspdf + qrcode added; QR pre-rendered to a PNG data URL on load so the
  print click stays in the user gesture (no popup-blocker trip).
- One PDF page per physical label: JOB# . WO#, description (bold, wrapped
  to 3 lines), finish (PAINT/COVER + set only), qty-overflow note, k/n.
- doc.autoPrint() + open blob → PDF print dialog; user picks GT800 at
  Actual size.
- HTML page is now preview-only; dead @media print rotation removed.

Orientation assumes the 50mm edge prints across the head (label reads
landscape). If the first physical print is 90 deg out, it's a one-line
flip of the PDF format/rotation — confirming on first print per request.
