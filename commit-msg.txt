Drop tbl_quotes.quote_value artefact + fix quoted total drift

Two related changes:

1. qry_job_accepted_quote was reading tbl_quotes.quote_value (header
   field) which is supposed to mirror SUM(tbl_quote_lines.line_value)
   but had no system maintaining the link. Result: 3 of 9 jobs had
   NULL headers despite having real line values, and the close report
   showed £0 quoted for those (Tramp, Projector). Tramp is what
   surfaced the bug — close dialog showed Quoted £0.00 instead of £600.

   View now sums directly from tbl_quote_lines for accepted quotes.
   Excludes auto-overhead lines (line_sub_group = 'Overhead') for
   consistency with the rest of the close report.

2. Dropped tbl_quotes.quote_value column entirely. Confirmed pure
   artefact via thorough sweep:
   - 0 app reads (no `.quote_value` accessor anywhere in src/)
   - 0 app writes (both INSERT paths into tbl_quotes — jobs/page.tsx
     and jobs/[id]/page.tsx — explicitly omit it)
   - 0 functions/triggers/constraints/RLS policies reference it
   - The only view consumer (qry_job_accepted_quote) was migrated to
     sum from lines

   Aligns with the S37 invariants discipline: single source of truth,
   no clever normalisation. Quote total is now derived from
   tbl_quote_lines always, with no parallel store to drift.

   Also dropped the qry_quote_value_drift watcher that was added in
   the prior migration — once the column is gone, the watcher has
   nothing to watch.

   Removed quote_value from the Quote TypeScript interface.

No app code changes beyond the one type-definition tweak.
