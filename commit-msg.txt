Fix React #310 in CostBreakdown onSummaryChange effect

The onSummaryChange useEffect was placed AFTER an early
`if (loading) return ...` statement. On first render loading===true
so the effect never ran. After data loaded, loading===false, the
component rendered past the early return, and the effect DID run
— meaning more hooks ran on subsequent renders than on the first,
which triggers React error #310 "Rendered more hooks than during
the previous render."

Moved the effect to right after the load() useEffect, before the
early return. Derived values (q, estTotal, committedTotal,
bestMarginPct, hasInvoiced) are computed inside the effect body
rather than referenced from scope below the early return, so the
hook call order is identical across all renders regardless of
loading state.
