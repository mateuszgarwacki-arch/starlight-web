Active workers banner: show both WO timers and quick timers

Problem: rpc_dashboard_data.activeWorkers only queried
tbl_wo_time_entries, so quick timers (tbl_tasks status=in_progress)
were invisible on the / dashboard banner. Two phantoms (Mateusz Filar
~81h, Radoslaw Kowalik ~405h) had been running unnoticed.

Fix (3 parts):

1. rpc_dashboard_data: activeWorkers now UNIONs WO timers and
   quick timers. Adds kind ('wo'|'task'), id, started_at,
   task_title. Sorted oldest-first so phantoms surface first.

2. Dashboard UI: each badge shows duration; colour escalates
   blue ≤12h, amber 12-24h, red >24h (phantom marker ⚠).
   Quick timers tagged " QT" so they're distinguishable from
   WO timers in the pill itself. Whole banner wraps in a Link
   to /workshop where stop controls live.

3. Phantom-timer patrol (new pg_cron job, daily 04:00 UTC):
   rpc_close_phantom_timers() auto-closes any timer running
   >16h. WO entries get system_end_timestamp=NOW(),
   actual_hours=0, flag_note explaining (surfaces in flags
   banner). Quick timers get status='pending', hours=0,
   review_note explaining (surfaces in PM inbox). Caps phantom
   accrual at ~24h max going forward.

Existing phantoms (Filar 81h, Kowalik 405h) will be caught by
tonight's run unless stopped manually first from /workshop.
