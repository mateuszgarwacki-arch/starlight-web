Promote-to-stock now actually promotes (immediate, dedup-aware)

Problem
- The "→ STOCK" toggle wrote `notes = 'PROMOTE_TO_STOCK'` as an intent
  flag. No trigger, cron, or completion hook ever acted on it, so items
  flagged for promotion sat as bespoke forever.
- Same for the "Add to stock catalogue when complete" checkbox on the
  bespoke dialog — it only set the same dead flag.

Fix
- New shared helper `src/lib/promote-to-stock.ts` creates a
  tbl_stock_items row immediately on click:
    * product_code = PROMO-{item_id}
    * stock_quantity = item.quantity
    * source_job_item_id = item.item_id (new FK)
    * hire_cost_day/_week = null (PM sets on Stock page)
  Then links the job item back via stock_item_id/stock_reference and
  sets item_source='promoted'. Clears the legacy PROMOTE_TO_STOCK flag
  only if present (never wipes real user notes).

- Dedup: if another item on the same job has already been promoted with
  the same exact description, merge into that stock row instead of
  creating a second one. Increments stock_quantity by the new item's qty.

- Scope page left-column "→ STOCK" button now calls the helper instead
  of the flag write.
- work-orders-panel.tsx addBespokeItem: if checkbox ticked, calls the
  helper immediately after insert. Checkbox label updated from "Add to
  stock catalogue when complete" → "Also add to stock catalogue" to
  reflect immediate behaviour.

Schema
- ALTER TABLE tbl_stock_items ADD COLUMN source_job_item_id INTEGER
  REFERENCES tbl_job_items(item_id) ON DELETE SET NULL.
- Index: idx_stock_items_source_job_item (partial, WHERE NOT NULL).

Backfill (job 8 / Grosvenor)
- 9 items flagged with PROMOTE_TO_STOCK → 7 deduplicated stock rows
  (PROMO-G-28, 29, 30, 38, 101, 102, 103). The three "Grosvenor Straight
  Bar, 6ft" entries (items 29/33/34) correctly merged into a single
  stock row qty 16.

Out of scope (follow-ups)
- job-items-table.tsx (dead code, not imported anywhere) not updated.
  Can be deleted in a later cleanup.
- Stock availability / build-state badge ("In build: 2/5 WOs complete")
  deferred to a follow-up session.
