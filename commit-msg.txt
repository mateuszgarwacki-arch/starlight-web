feat(load-list): truck grouping + bulk assign + per-truck print

Minimal multi-truck support for the load list. Supplements — does
not replace — the existing load documentation system.

Schema:
- tbl_load_groups: per-job named containers ("Truck 1 — Main",
  "Client collection", etc.) with optional driver + departure time.
  Admin/manager/PM create/edit/delete; foreman read-only.
- tbl_load_events.load_group_id: nullable FK to tbl_load_groups
  (NULL = unassigned). On group delete, items fall back to NULL
  via ON DELETE SET NULL.
- rpc_load_list now returns { job, groups, items } — items include
  load_group_id so the UI can render Zone → Truck → Items.

UI (rewrite of /reports/load-list/[jobId]):
- Truck manager strip: add/delete trucks, per-truck print button,
  shows name/driver/departure/progress (loaded/total).
- Bulk selection via row checkboxes. Floating "N selected" action
  bar (sticky top) with buttons to assign to any truck or unassign.
  Bulk upsert to tbl_load_events preserves existing status/stamps.
- On-screen grouping: Zone → Truck → Items (Unassigned bucket
  appears first within each zone if populated). This gives PMs
  the cross-job-zone overview requested.
- Print modes:
  * "Print everything" (header): all trucks + unassigned, one per page
  * "Print all trucks" (when groups exist): same as above, explicit
  * Per-truck print icon: single truck only
  Each printed page: truck header (name/driver/departure), then
  zones within as subsections, then signature block (Packed by /
  Loaded by). break-before-page between trucks, break-inside-avoid
  on zone blocks and signature.
