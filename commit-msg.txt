S51 fix: drop break-inside-avoid on MaterialCutPlan wrapper

Empty cut-plan page on long cut lists (e.g. job 13585 step 2 of 2,
33 sheets of OSB + 4 sheets of plywood).

Root cause: print CSS

  .traveller-page.traveller-cover { page-break-inside: auto !important }

correctly lets the cut plan page spill across printed pages, but inside
the page MaterialCutPlan wrapped its whole block in break-inside-avoid.
For a ~770mm material block, Chromium honoured the avoid hint by pushing
the whole block past the current page — landing nowhere it actually fit,
leaving the cut-plan page rendering only the section header.

Fix: remove break-inside-avoid from the material wrapper; keep per-sheet
break-inside-avoid on each SheetLayout (those are ~44mm, satisfiable);
add break-after-avoid to the material header paragraph so the header
doesn't orphan from the first sheet.
