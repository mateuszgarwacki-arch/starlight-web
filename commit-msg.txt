assignee chip picker. Peer model: all assignees are equal, no
hierarchy. Mobile "Lead" filter renamed to "On" and now matches
any WO where the user is an assignee (not just first/lead).

  assigned_by) with UNIQUE(wo, freelancer). RLS: all authed users

  can INSERT/DELETE.
- Sync trigger trg_sync_planned_lead_from_assignees: on any insert

  assignee (by assigned_at, tiebreak wo_assignee_id) or NULL if
  none. This keeps existing workshop/capacity/traveller pages
  working unchanged — they still read planned_lead_id.
- Backfill: seeded tbl_wo_assignees from existing planned_lead_id
  values (4 rows migrated, verified).
