Handover: rotate drawings feature (port of traveller pattern)

Same rotation pattern the traveller print surface uses:
- Per-drawing rotation state in HandoverPage. Cycles 0→90→180→270→0.
- Rotate button (small, next to the caption) with "print:hidden" so
  it shows on screen but doesn't appear on the printed page.
- Auto-detect landscape images on load and rotate 90° once so they
  fill the portrait page. PM can override manually after.
- Rotation key scoped per zone-drawing pair (event_zone:doc_id) so
  the same document attached to multiple zones rotates independently.
- Swaps max-width/max-height at 90°/270° so rotated images don't
  overflow the A4 page frame.

Not persisted — per-session only. Matches traveller behaviour.
Good enough for the "oops, that drawing is sideways" case; if you
later want the rotation to stick across sessions, that's a small
follow-up (add rotation to tbl_handover_zone_documents).
