S38e: Kit list — include scope-level BOM materials

Scope-level materials added to tbl_wo_bom (e.g. 3x2 timbers put directly
on a scope for on-site use) were invisible on the traveller pack-mode
kit list because pack-mode fetched only tbl_job_items. Site-bound
materials must be on the pack list — end of.

Fix (traveller/page.tsx):
- Pack-mode now loads both tbl_job_items (deliverables) AND tbl_wo_bom
  (scope-level materials where work_order_id IS NULL AND job_item_id IS NULL)
- BOM rows merged in as synthetic ScopeJobItem entries (negative item_id)
  so ScopeOverview renders them without changes
- rpc_load_list already includes these rows; no RPC change needed

No schema change.
