Allow routing time entries to Complete WOs

Three surfaces previously filtered to active statuses only and would not
let a PM route a late-arriving ad-hoc task (or manually log time) to a
work order that had already been marked Complete:

- src/components/route-task-modal.tsx (used on /review/inbox)
- src/app/(dashboard)/crew/[id]/page.tsx openRouteModal
- src/app/(dashboard)/crew/[id]/page.tsx openAddEntry

All three now include "Complete" in the WO status filter. Within each
scope group, active WOs sort first and Complete sort to the bottom.
Complete rows render with opacity-60 and the existing badge-complete
status pill (via statusClass) makes them visually distinct.

The crew page's two inline lists previously hardcoded a ternary that
only highlighted In-Progress; both now use statusClass() for consistent
badge styling, which gives Complete its proper pill for free.

No schema changes. Freelancer mobile flows (m/me, m/task, m/page) still
filter active-only by design — only PMs route time, freelancers should
not be able to log new hours against finished work without PM review.
