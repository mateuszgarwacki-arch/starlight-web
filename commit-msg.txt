S38g: Edit time-entry date from by-day view + preserve time-of-day

Mateusz could not edit a time entry's date in the new default by-day
view — a freelancer put hours on the wrong day and there was no date
field in the inline edit. Fixed two problems:

1. By-day edit UI (crew/[id]/page.tsx):
   Added a date + flag panel below the row when editing. Same pattern
   as the archive-reason panel. Log view already had this; now parity.

2. handleEditEntry logic:
   Old behaviour hard-coded "T09:00:00" on date change and never
   touched system_end_timestamp — so moving an entry across days would
   reset the start time and leave the end timestamp stranded on the
   original day, breaking duration and ordering.

   New behaviour: preserve original time-of-day on start, compute the
   date delta, shift end by the same delta. Sets timestamp_edited_flag
   so the "edited" badge surfaces. No Z suffix on the timestamp strings
   to avoid BST date-shift. If the entry has no original end (open
   timer), skip the end update rather than invent one.
