Handover: strip ceremony + fix misleading kit-list readiness

Four fixes based on live feedback from Grosvenor's first handover:

1. Cover header: strip "Handover Summary" + subtitle + Starlight
   eyebrow. Job number + name with the bordered frame is
   self-evident. Less paperwork ceremony, more info density.

2. Remove client name from cover. Starlight's clients value
   discretion — their names don't belong on internal paperwork
   circulating around the workshop.

3. Kit-list readiness was misleading. `kit_list_exported=true`
   means "on the truck plan", which happens *after* items are
   prep-ready. Showing "0 of 89 items kit-listed" implied the
   workshop is nowhere near ready when in fact they're fully
   prep-ready — the truck plan just hasn't been made yet.
   Replaced with a neutral "89 items" count on the readiness
   strip. Kit column on the per-item table removed for the
   same reason — CircleDashed amber icons on every row told
   the wrong story.

4. Removed the "No drawings attached to this zone. Attach them
   from the job page when back at the office." empty-state
   message. Zones without drawings now just show nothing.
   Until the Phase 3 authoring UI ships, the message is just
   noise on every page.

Also drops the now-unused Briefcase lucide import.

No schema change. Data layer unchanged.
