fix(jobs): show live accepted-quote total as "Quoted" instead of dead budget_allowance

The Jobs list "Budget" column was reading tbl_production_plan.budget_allowance,
which is a fossil field — the app reads it in 5 places but never writes it.
Values were populated once during the original Access migration and have been
stale ever since. After updating job 13757's quote from £157,515 to £180,265,
the list still showed £157,515 because budget_allowance never got the memo.

Fix: column header "Budget" → "Quoted". Loader now joins
qry_job_accepted_quote and shows accepted_quote_value, so the list always
reflects the live commercial number that's already shown inside each job.

The budget_allowance field is left in place — it can later become a real
"client-stated budget" concept with proper UI for variance tracking, but
that's a separate feature. The other 4 read-sites (PM jobs, review,
handover, job-close) keep using it for now and can be migrated piecemeal.
