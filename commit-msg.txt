fix(jobs): scroll-restore actually works (S50c)

S50a shipped the feature but it never landed scroll on prod -
Mateusz tested and got "lands at top, ignores hash entirely".

Two bugs, both fixed:

1. Next.js App Router scroll-to-top winning the race against
   scrollIntoView. Added scroll={false} to the scope page's
   Back-to-Job Link so Next.js doesn't auto-scroll after nav.

2. didRestoreScroll ref was set to true BEFORE the rAF callback
   ran, so if the row wasn't in the DOM yet (data still loading)
   the flag burnt and we never retried. New effect:
   - Only marks done after scrollIntoView actually runs
   - Re-fires on woData.length / scopes.length / activeTab changes
   - setTimeout(50ms) instead of rAF for slacker timing
   - behavior: "auto" instead of "instant" for broader support
