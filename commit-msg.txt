fix(labels): portrait page + 90° rotated content for the GT800 (S53)

The 50x25mm PDF seated correctly (one label per label — page size was
right), but printed 90° rotated: the GT800 feeds this stock portrait and
rotates a landscape page to fit. Now we author a portrait 25x50mm page
and render each label's landscape content rotated 90° onto a canvas, so
the page matches the printer's feed and prints upright, reading along the
50mm edge. Footprint still equals the label, so seating is preserved.

- generatePdf now renders via an offscreen canvas (rotated 90° CW) and
  adds one full-page PNG per label; QR pre-decoded to an <img> so the
  print click stays in the user gesture.
- If a print ever comes out 180° (upside down), it's a one-line swap of
  the canvas rotate direction (noted in code).
