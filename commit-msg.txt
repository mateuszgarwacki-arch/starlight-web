S36c job page: filter persistence, scope pill, toolbar row

- Quote-line filter now persists per-job in sessionStorage
  (survives drill-into-scope + back navigation; dies on tab close)
- Scope column on quote-line rows now shows a coloured status pill
  ("Active" blue / "Provisional" amber / "Built" green / "Cancelled" grey)
  instead of a plain FileText icon. Pill is the link.
- Scope status values added to statusClass() in lib/utils.ts:
  Active -> in-progress, Provisional -> on-hold,
  Workshop Completed -> complete, Cancelled-Cost-Retained -> voided
- Five stacked panels (PM Queries, Learnings, Invoices, Orders,
  Job Documents) replaced with a single toolbar row + expand-on-click.
  Cost Analysis stays full-width above.
- New RPC rpc_job_header_counts returns counts + invoice unallocated
  total in one call, parallel to existing rpc_job_detail_data.
- Button badges: amber warning badge when action needed
  (open queries, outstanding orders, unallocated invoices, open learnings).
- PmQueriesJobPanel, JobInvoicesPanel, JobOrdersPanel accept new
  defaultExpanded/defaultCollapsed prop so toolbar can force open.
