S33: WODocumentsPanel -> auditedInsert + auditedDelete

Closes the Phase 2 audit gap on tbl_wo_documents. Uploads and deletions
now appear in Settings -> Audit Log.

- Added tbl_wo_documents: "doc_id" to AUDITED_TABLES registry in
  src/lib/audit.ts so auditedInsert/Delete actually write to the log
  (both functions check AUDITED_TABLES[tableName] before logging)
- Converted handleUpload's insert to auditedInsert with ctx + jobId
- Converted deleteDoc's delete to auditedDelete with ctx + jobId
- Left the drag-reorder sort_order update as raw supabase.update().
  Reordering is cosmetic and would generate one audit row per doc per
  drag, which is noise rather than signal.

Value: uploads already captured uploaded_by + uploaded_at on the row
itself, so auditedInsert adds an explicit "insert" log entry (visible
alongside scope/WO creation). auditedDelete is the more valuable change
- previously a deleted doc left zero trace in the system. Now the
deletion is logged with a full snapshot in old_value so a PM can see
what was removed and by whom.
