feat(mobile): Lead filter on tasks tab for planned-lead WOs

Adds a distinct "Lead (N)" filter pill to the mobile Tasks screen
alongside the existing All/Mine/Done/Painting pills.

Rationale: previously the only way to see WOs you'd been assigned
as planned_lead_id was to have already started a timer or clocked
time against them. This is fine for freelancers but doesn't match
the mental model of a PM or foreman who plans work ahead.

"Mine" keeps its existing meaning (hands-on: open timer OR logged
time) to avoid bloating it with not-yet-started WOs.
"Lead" is a new, separate concept showing WOs where the current
user is the planned lead, regardless of timer state.

The Lead pill:
- Only appears when the user actually has >=1 assigned WO, so
  freelancers who never get set as lead never see it.
- Uses the starlight-blue accent (not navy) to visually distinguish
  intent/planning from action.
- UserCheck icon + count so it reads as "what I'm expected to run".

Implementation:
- Added planned_lead_id to both tbl_work_orders selects
  (active Ready/In-Progress + Complete WOs from active jobs).
- Added planned_lead_id to TaskCard interface.
- Populated planned_lead_id when building TaskCards.
- Extended filter type to include "assigned".
- Added assignedTasks = activeTasks.filter(t => t.planned_lead_id === myId).
- Added pill button (conditional on assignedTasks.length > 0).

No schema changes, no RPC changes — planned_lead_id already existed
and was already being populated via the desktop WO assignment UI.
