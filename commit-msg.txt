feat(wo-docs): view cutlists inline instead of forcing download

Cutlists in the work order docs panel previously only had Download and
Remove. Mateusz wanted to view them in-browser. Adds an Eye button on
desktop and makes tapping the cutlist name open inline on mobile.

Why a new API route rather than reusing /api/onedrive/download:
- /api/onedrive/download returns Microsoft's signed CDN URL, which has
  Content-Disposition: attachment baked in by OneDrive — the browser
  forces download regardless of mime type.
- New /api/onedrive/view fetches the file server-side and re-streams it
  with Content-Disposition: inline + the original mime type. PDFs and
  images render in the new tab; unsupported types fall back to download
  via browser default behaviour.

Auth and security:
- Same Supabase-session trust boundary as /api/onedrive/download.
- Accepts auth via Authorization header OR ?token= query param. Query
  param is needed because window.open() opens a fresh tab and can't set
  request headers. The token is the user's own short-lived (1h) Supabase
  access token — leakage risk is bounded to that window, and it's the
  same trust as any logged-in API call.
- Body is streamed (Response(fetchRes.body)) so files of any size work
  inside Vercel's function limits without buffering.

Mobile UX: tap on the cutlist name now opens View; Share and Download
icons retained on the right. This matches the natural mobile gesture
(tap-to-open) that workshop staff expect.
