S38f: Crew timeline — one unified view (by-day default, Log alternate)

Mateusz wanted a single view of a crew member's time entries that
shows everything recorded without switching to the Flat tab.

Changes (crew/[id]/page.tsx):
- Default viewMode: "flat" → "by-day"
- Renamed "Flat" → "Log" (internal key + button label)
- By-day sort: entries without a date now float to the TOP instead of
  being buried at the bottom (unknown-first, then reverse chronological)
- Loaded previously-missing fields: tbl_wo_time_entries.actual_start_timestamp,
  actual_end_timestamp, timestamp_edited_flag; tbl_tasks.photo_urls,
  started_at
- By-day rows gained a detail strip rendered inline (no extra expand):
    WO rows:    start→end times + "edited" badge + full flag note text
                + archive reason when applicable
    Task rows:  start time + category + description + photo thumbnails
                + PM review note
  Detail strip omitted entirely when there's nothing to show.
- Log (Flat) view gained: photo thumbnails on task cards + "edited" badge
  on WO cards when timestamps were manually adjusted.

Photos are Sharepoint URLs and require the viewer to be signed into 365
in the same browser session. Broken-image handler falls back to a placeholder.
